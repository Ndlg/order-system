# 订单整理系统

面单采集数据标准化与档口报货 Excel 生成系统。

## 项目定位

本项目用于把面单采集工具返回的原始采集数据，经过标准化处理后，生成可直接发给档口的报货 Excel。

核心链路：

```text
采集工具 / 标准化 Excel
  ↓
原始采集记录留存
  ↓
面单模板解析
  ↓
标准化明细
  ↓
字段角色配置
  ↓
鞋款简称映射
  ↓
规格 / 图片 / 档口映射
  ↓
报货汇总
  ↓
三种方式导出 Excel
```

## 核心原则

1. 系统最终处理标准化明细，不直接绑定原始采集格式。
2. 原始采集内容统一称为 `raw_payload`，不得假设一定存在 `raw_json`。
3. 标准化明细第一版字段固定，后续再扩展配置能力。
4. 店铺字段、鞋款简称字段、规格字段、尺码字段、数量字段均由用户配置。
5. 鞋款简称映射、规格映射、图片绑定、档口归属均由用户维护。
6. 禁止把具体店铺、鞋款、规格、面单字段、导出字段写死在代码中。
7. 最终报货 Excel 必须支持：合并 Sheet、鞋款档口 Sheet、鞋款档口文档。
8. 支持 Windows 内网服务器部署。

## 技术栈

- 前端：Vue 3 + TypeScript + Element Plus
- 后端：Python FastAPI
- 数据库：MySQL
- 缓存：Redis
- Excel：openpyxl
- 图片处理：Pillow
- 部署：Windows 优先，后续兼容 Docker
