# 订单整理系统

面单采集数据标准化与档口报货 Excel 生成系统。

## 项目定位

本项目用于把面单采集工具返回的原始采集数据，经过面单模式解析和标准化处理后，生成可直接发给档口的报货 Excel。

本系统不是店铺订单系统，也不是按店铺整理订单。系统只按面单信息、面单模式和用户配置进行处理。

店铺名、淘宝店铺名、抖店店铺名等，只能作为标准化明细里的普通字段存在，由用户在字段角色配置中自行定义，不能作为系统一级业务对象。

核心链路：

```text
采集工具 / 标准化 Excel
  ↓
原始采集记录留存
  ↓
面单模式识别
  ↓
面单模板解析
  ↓
标准化明细
  ↓
字段角色配置
  ↓
鞋款简称映射
  ↓
规格 / 图片 / 档口映射
  ↓
报货汇总
  ↓
三种方式导出 Excel
```

## 核心原则

1. 系统根据面单信息和面单模式整理数据，不根据店铺整理订单。
2. 店铺名只是标准化明细字段，是否使用、如何使用，由用户配置。
3. 系统最终处理标准化明细，不直接绑定原始采集格式。
4. 原始采集内容统一称为 `raw_payload`，不得假设一定存在 `raw_json`。
5. 标准化明细第一版字段固定，后续再扩展配置能力。
6. 面单模式字段、来源店铺字段、鞋款简称字段、规格字段、尺码字段、数量字段均由用户配置。
7. 鞋款简称映射、规格映射、图片绑定、档口归属均由用户按工作空间维护。
8. 禁止把具体店铺、鞋款、规格、面单字段、导出字段写死在代码中。
9. 最终报货 Excel 必须支持：合并 Sheet、鞋款档口 Sheet、鞋款档口文档。
10. 支持 Windows 内网服务器部署。

## 技术栈

- 前端：Vue 3 + TypeScript + Element Plus
- 后端：Python FastAPI
- 数据库：MySQL
- 缓存：Redis
- Excel：openpyxl
- 图片处理：Pillow
- 部署：Windows 优先，后续兼容 Docker
