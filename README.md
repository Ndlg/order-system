# 订单整理系统

面单模式驱动的可配置报货 Excel 平台。

## 项目定位

本系统用于接收采集工具或用户上传的数据，识别面单模式，生成标准化明细，再根据用户在 Web 界面维护的配置生成报货 Excel。

系统要做的是平台框架，不是业务规则定制。

系统只负责：

```text
面单模式识别
原始采集内容留存
面单模板解析
标准化明细管理
字段角色配置
映射关系维护
图片绑定管理
档口资料维护
报货汇总引擎
Excel 导出引擎
异常提示与人工修正入口
多用户工作空间隔离
```

系统不负责在代码里定义：

```text
哪个字段一定是店铺
哪个字段一定是鞋款简称
哪个简称对应哪个鞋款
哪个规格对应哪个标准规格
哪个鞋款归哪个档口
哪个规格用哪张图片
最终 Excel 必须按哪一种业务固定格式输出
```

这些全部由用户在 Web 界面配置。

## 最高原则

```text
1. 系统不是店铺订单系统。
2. 系统不是按店铺整理订单。
3. 系统只按面单信息、面单模式、标准化明细和用户配置处理数据。
4. 店铺名、淘宝店铺名、抖店店铺名等，只能作为标准化明细中的普通字段存在。
5. 店铺字段是否使用、如何使用，全部由用户在字段角色配置中定义。
6. 不同用户维护不同规则、鞋款定义、规格映射、图片绑定、档口归属和导出配置。
7. 这些配置必须按 workspace_id 隔离。
8. 具体业务关系全部由用户定义，不允许写死在代码中。
```

一句话：

```text
系统只识别和处理面单模式，其他业务规则都交给用户定义。
```

## 核心链路

```text
采集工具 / 标准化 Excel
  ↓
原始采集记录留存
  ↓
面单模式识别
  ↓
面单模板解析
  ↓
标准化明细
  ↓
字段角色配置
  ↓
用户定义映射关系
  ↓
用户定义图片 / 档口 / 导出规则
  ↓
报货汇总
  ↓
三种方式导出 Excel
```

## 技术栈

- 前端：Vue 3 + TypeScript + Element Plus
- 后端：Python FastAPI
- 数据库：MySQL
- 缓存：Redis
- Excel：openpyxl
- 图片处理：Pillow
- 部署：Windows 优先，后续兼容 Docker
