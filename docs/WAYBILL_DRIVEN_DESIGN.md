# 面单驱动设计原则

> 本文档是辅助原则文档。如与 `docs/CODEX_TASKBOOK.md` 冲突，以 `docs/CODEX_TASKBOOK.md` 为准。

## 核心结论

订单整理系统不是根据店铺整理订单，而是根据面单信息、面单模式、标准化明细和用户配置整理数据。

店铺名、淘宝店铺名、抖店店铺名等，只是面单信息或标准化明细中可能出现的普通字段。系统不能把店铺作为一级业务对象，也不能把店铺作为整理主线。

## 正确主线

```text
原始采集数据 / 标准化 Excel
  ↓
识别面单模式
  ↓
匹配面单模板
  ↓
生成标准化明细
  ↓
用户配置字段角色
  ↓
用户配置业务简称 / 鞋款简称映射
  ↓
用户配置规格 / 图片 / 档口 / 导出方式
  ↓
报货汇总
  ↓
Excel 导出
```

系统只负责提供这条链路的能力，不负责内置具体业务规则。

## 面单模式是核心入口

系统首先关心的是：

```text
这条采集数据属于哪种面单模式？
```

面单模式决定：

```text
使用哪个面单模板
如何把原始采集内容转换成标准化明细
使用哪个工作空间下的配置
```

面单模式不代表店铺，也不等于平台来源。它只是用户定义的一种数据结构识别方式。

## 字段角色由用户定义

标准化明细第一版默认字段：

```text
面单模式
完整原文
店铺名
商品简称
规格
尺码
数量
备注
图片识别
任务ID
文档ID
任务时间
采集端ID
来源机器
来源序号
原始内容
```

这些字段只是标准化明细列。它们的业务含义由用户配置。

示例：

```text
面单模式字段 = 面单模式
来源店铺字段 = 店铺名
业务简称字段 = 商品简称
规格字段 = 规格
尺码字段 = 尺码
数量字段 = 数量
备注字段 = 备注
```

其中 `来源店铺字段` 是可选普通字段，只能用于展示、筛选、导出或用户自定义分组，不能触发系统级店铺模型。

## 用户配置决定业务结果

以下内容必须由用户配置：

```text
哪个字段作为业务简称 / 鞋款简称
来源简称映射成哪个标准分类 / 标准鞋款
原始规格映射成哪个标准规格
哪个标准分类 / 标准规格绑定哪张图片
哪个标准分类 / 标准规格归属哪个档口
最终 Excel 按什么字段分组
最终 Excel 采用哪种导出方式
```

代码只执行配置，不写死规则。

## 禁止事项

```text
禁止根据店铺创建系统一级对象。
禁止根据店铺决定订单整理流程。
禁止使用 shop_id 作为核心隔离字段。
禁止创建 shops 表作为业务主表。
禁止把店铺名写入业务规则代码。
禁止让鞋款规则、规格规则、图片规则跟店铺强绑定。
禁止把店铺字段设计为必填字段。
禁止在代码里内置具体鞋款、规格、档口、图片关系。
```

## 后续扩展

如果用户希望按标准化明细中的店铺名筛选、分组或拆分报表，可以作为字段角色和导出配置实现。

这仍然属于用户配置层，不改变系统的面单驱动主线。
