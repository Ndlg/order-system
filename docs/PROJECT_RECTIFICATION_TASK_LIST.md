
### 4. 商品/SKU 识别规则收紧

```text

[ ] 多候选商品或 SKU 时进入冲突，不自动选。
[ ] 备注中明确写规格/尺码时覆盖销售属性1/2。
```

## 每日执行顺序

```text
1. 先确认 Docker 服务和采集器在线状态。
2. 再跑当前 P0 测试组。
3. 只挑一个 P0 主题整改。
4. 整改后立刻补测试或样本。
5. 重建对应服务时使用最小范围命令。
6. 更新本清单和阶段任务书状态。
```

## 禁止事项

```text
禁止无备份清库。
禁止无范围全库重解析。
禁止在业务机上继续堆 bat/vbs 作为正式启动方式。
禁止为了消除异常而把冲突自动算作已匹配。
禁止把现场测试 IP 写入通用采集器文档或默认配置。
```
