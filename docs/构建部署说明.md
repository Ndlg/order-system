# 构建部署说明

## 项目定位

本仓库是订单整理系统源码仓库，适合提交到 GitHub 保存和后续迭代。

源码目录只保存程序源码、配置、测试、文档和构建脚本，不保存用户业务数据。

## 源码目录

将仓库放到任意固定目录即可，下面统一用占位符表示：

```text
<PROJECT_DIR>
```

后续命令都在该目录下执行。

## 用户数据位置

用户数据不在源码目录里。

默认 Docker 数据卷：

```text
order-system-data
```

数据卷中包含：

```text
/data/order-system.db
/data/workspaces
```

这些是数据库、上传图片、采集面单、导出文件等业务数据。

提交 GitHub 或拷贝源码时，不要把 Docker 数据卷内容放进仓库。

## Docker 镜像说明

镜像是由源码构建出来的可运行程序包。

默认镜像：

```text
order-system-backend:latest
order-system-tenant-ui:latest
order-system-platform-admin-ui:latest
redis:7-alpine
```

关系：

```text
源码 + Dockerfile + docker-compose 配置
  -> docker compose build
  -> Docker 镜像
  -> docker compose up
  -> Docker 容器
  -> 读写 Docker 数据卷
```

## 构建前准备

需要安装：

```text
Docker Desktop
Git
```

首次运行前创建数据卷：

```powershell
docker volume create order-system-data
```

如果是已经投入使用的机器，这个数据卷通常已经存在，不要删除。

## 只构建镜像

进入项目目录：

```powershell
cd <PROJECT_DIR>
```

构建镜像：

```powershell
docker compose -p order-system -f docker-compose.yml -f docker-compose.site.yml build backend tenant-ui platform-admin-ui
```

该命令只构建镜像，不启动服务。

## 启动服务

```powershell
docker compose -p order-system -f docker-compose.yml -f docker-compose.site.yml up -d
```

检查状态：

```powershell
docker compose -p order-system -f docker-compose.yml -f docker-compose.site.yml ps
```

正常应看到：

```text
order-system-backend            Up
order-system-tenant-ui          Up
order-system-platform-admin-ui  Up
order-system-redis              Up
```

## 停止服务

停止容器但保留数据：

```powershell
docker compose -p order-system -f docker-compose.yml -f docker-compose.site.yml down
```

不要执行：

```powershell
docker compose down -v
```

`-v` 会删除数据卷，可能导致业务数据丢失。

## GitHub 提交流程

检查远端：

```powershell
git remote -v
```

如果没有远端，需要先添加新仓库地址：

```powershell
git remote add origin https://github.com/<owner>/<repo>.git
```

提交前检查：

```powershell
git status --short
```

不要提交：

```text
storage 里的业务数据
*.db
*.sqlite
*.log
frontend/node_modules
frontend/dist
frontend/dist-server-admin
.venv
.pytest_cache
```

提交示例：

```powershell
git add .
git commit -m "Prepare order system deployment baseline"
git push -u origin main
```

## 源码清理检查

上传前建议再检查一遍：

```powershell
rg -n -i "个人姓名|现场IP|本机用户名|业务数据关键字" .
git status --short
```

源码仓库应只包含可构建项目，不包含客户业务数据、现场机器信息或个人目录。
