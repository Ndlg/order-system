# 阶段 21：客户侧商品识别流程纠偏

> 阶段定位：移除客户管理页中无效的“字段用途 / 关键字段”入口，把客户侧配置收敛到“商品识别”。服务端提供面单解析能力，业务端使用解析出来的文字先识别商品，SKU 后续由系统按商品下的 SKU 自动匹配。

## 1. 本阶段纠偏

错误方向：

```text
客户先进入字段用途
再进入关键字段
再进入匹配规则
```

问题：

```text
字段和关键字段并不是客户真正关心的业务动作。
平台解析层已经提供了可读字段。
客户业务端要做的是：使用这些解析结果去识别商品。
```

正确方向：

```text
服务端解析面单
  ↓
生成可读的商品文字、规格文字、数量等结果
  ↓
客户管理页进入商品识别
  ↓
设置关键词如何关联到商品
  ↓
后续批次自动识别商品 / SKU / 图片 / 档口
```

## 2. 已完成调整

```text
客户管理页移除“字段用途”入口。
客户管理页移除“关键字段”入口。
旧地址 /admin/field-definition 和 /admin/key-fields 自动跳转到 /admin/matching。
“匹配规则”改名为“商品识别”。
商品识别页不再展示字段说明表，客户直接输入面单上看到的商品关键词。
商品识别页不再展示关键词命中统计，避免客户误解为必须依赖系统候选字段。
新增规则时，客户输入识别关键词并关联到商品，不再人工指定 SKU。
SKU 由系统使用面单商品文字与该商品下的 SKU 名称自动匹配。
系统内部仍用 key_field_sets 记录平台解析商品字段，但不再暴露给客户操作。
```

## 3. 当前商品识别字段

```text
product_full_text：商品信息
spec_text：规格文本
quantity：数量
```

业务侧重点：

```text
客户不维护这些字段本身。
客户只看这些字段提供了什么文字。
客户基于这些文字配置商品关联规则，SKU 匹配由系统自动完成。
```
