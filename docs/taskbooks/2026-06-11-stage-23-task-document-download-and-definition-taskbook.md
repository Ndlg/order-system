# 阶段 23：采集任务文档下载与表头定义

> 阶段定位：同一轮采集任务需要提供两份下载文档：采集原文文档、识别/整理后文档。同时，整理后文档的表头不能由平台固定，必须由客户工作区定义。

## 1. 本阶段目标

```text
为采集任务提供原文 Excel 下载。
为采集任务提供整理结果 Excel 下载。
整理结果 Excel 不固定系统表头。
客户管理员可以保存当前工作区的整理文档表头定义。
下载整理文档时按当前工作区已保存表头输出。
```

## 2. 已完成

```text
新增 GET /api/v1/collector-control/tasks/{task_id}/raw-document。
新增 GET /api/v1/collector-control/tasks/{task_id}/standard-document。
采集记录页的最近采集任务增加“原文 / 整理”下载按钮。
面单批次页当前任务增加“下载原文 / 下载整理文档”按钮。
识别预览页增加可编辑的导出表头定义草稿。
客户点击“保存为当前工作区表头”后，定义写入当前 workspace。
整理文档下载只读取当前 workspace 已保存且启用的表头定义。
如果未定义表头，整理文档只输出提示，不使用平台固定列。
```

## 3. 当前表头定义方式

```text
表头名称：客户自己填写，例如 商品简称、规格、尺码、数量。
取值来源：客户从平台解析结果或计算字段中选择，例如 product_display_text、spec_text、inferred_size、quantity。
导出顺序：按保存草稿时的顺序写入 export_order。
生效范围：当前 workspace。
```

## 4. 当前边界

```text
原文文档结构仍由平台提供，因为它用于诊断采集器回传内容。
整理文档表头已经不固定在系统中，但目前定义入口还在“识别预览”页内。
识别步骤的完整可配置化尚未完成，目前商品识别仍使用商品识别规则，SKU 匹配仍是最小自动匹配逻辑。
后续需要把“表头定义”和“识别步骤定义”整理成独立客户配置页。
```
