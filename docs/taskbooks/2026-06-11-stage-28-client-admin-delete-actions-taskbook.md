# 阶段 28：客户管理端删除操作补齐

## 背景

客户管理端可以新增商品、SKU、商品识别规则和采集器连接，但列表缺少删除按钮。

这会导致测试和真实使用中产生大量错误配置，无法由用户自行清理。

## 目标

```text
当前抖店第一版客户管理页补齐删除/移除操作。
删除操作必须有确认弹窗。
删除走后端软删除接口，不做前端假删除。
```

## 已完成

```text
新增前端 deleteRecord 通用 API。
商品识别规则列表增加删除按钮。
商品列表增加删除按钮。
SKU 列表增加删除按钮。
采集连接列表增加移除按钮。
所有删除/移除操作增加确认弹窗。
删除成功后自动刷新当前列表。
```

## 当前口径

```text
删除是软删除，避免直接物理清库。
商品删除会从当前商品列表移除。
SKU 删除会从当前商品 SKU 列表移除。
商品识别规则删除后不再参与后续匹配。
采集器移除后，业务机重新登录可以再次连接。
```
