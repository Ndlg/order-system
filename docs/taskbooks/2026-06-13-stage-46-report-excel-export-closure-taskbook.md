# 阶段 46：报货 Excel 导出闭环

> 日期：2026-06-13  
> 状态：已完成本轮第一版接入

## 阶段定位

前面阶段已经把真实采集、我打模板归类、字段 key 定义、商品主类识别和 SKU 自动匹配跑通到预览层。现在进入导出阶段，目标不是继续下载“整理字段表”，而是生成业务现场真正能使用的报货 Excel。

## 当前基础

```text
1. 监听批次已经能归档读取。
2. 我打面单已经按 printXML 版式指纹命中客户模板。
3. 单商品和多商品已经统一展开为商品项列表。
4. 商品识别预览已经能输出商品、SKU、数量、状态和异常原因。
5. 当前已有“下载原文”和“下载整理文档”，但整理文档只按字段表头导出，不等于最终报货单。
```

## 本阶段目标

```text
新增报货 Excel 下载能力。
导出中心可以选择监听批次。
导出数据以商品识别结果为准，而不是只导出 standard_details 原字段。
一张面单多个商品时，每个商品项输出一行。
单商品面单也按一个商品项输出一行，和多商品共用同一套导出逻辑。
Excel 中明确展示匹配状态，未命中和冲突不能静默当成成功。
第一版先生成无图片报货表，图片嵌入和复杂分组放到后续阶段。
```

## 本轮完成

```text
新增 GET /api/v1/collector-control/tasks/{task_id}/recognition-report。
报货 Excel 复用商品识别执行服务，不另起一套匹配逻辑。
报货 Excel 一行对应一个商品项，多商品面单按商品项展开。
导出列包含面单、商品项、面单模板、商品文字、销售属性1、销售属性2、数量、识别商品、识别 SKU、匹配状态、命中方式、命中字段、命中关键词、异常原因和备注字段。
导出中心从占位页改为可用页面：选择监听批次、查看识别汇总、预览报货结果、下载报货 Excel。
新增后端测试覆盖已匹配和未匹配商品项导出。
```

## 追加纠偏：混合批次与 SKU 报货图

```text
一次监听批次可能同时包含我打、菜鸟/淘宝直打、抖店直打。
正式报货 Excel 不能按面单来源拆开导出，必须按当前批次全部商品识别结果统一导出。
导出中心的报货 Excel 增加报货图列。
报货图来自当前商品行识别命中的 SKU 绑定图片。
如果 SKU 未绑定图片，不阻断导出，Excel 图片列留空。
如果商品或 SKU 未命中，仍导出该行并保留异常原因。
抖店和菜鸟直打的 SKU 自动匹配补充读取商品简称、规格文本、商品信息、买家备注和卖家备注，不再只依赖我打销售属性字段。
导出中心预览增加报货图绑定状态。
```

## 追加完成：SKU 图片预览

```text
导出中心的报货结果预览中，报货图列直接展示 SKU 图片缩略图。
缩略图读取当前商品行识别命中的 sku_image_asset_id。
如果 SKU 已命中但没有绑定图片，则显示“未绑定”。
Excel 下载仍然使用同一个 sku_image_asset_id 嵌入图片。
```

## 追加修正：导出模板替换语义

```text
原“套用”预设会把多个预设字段叠加到同一个工作区表头，导致预览出现重复和混乱列。
按钮文案从“套用”改为“替换当前表头”。
点击导出模板时，先确认，再清空当前已定义表头，最后写入所选模板字段。
用户仍可在替换后继续手动编辑表头名称、字段和排序。
```

```text

内部复用商品识别执行服务 recognize_detail_items。
按当前工作区读取该批次 standard_details。

生成 rows 后直接写入 xlsx。

```

## 追加修正：导出表头页面降复杂度

```text
用户反馈导出表头页面信息太多，业务现场不需要默认理解字段字典和系统 key。
导出表头页不再提供导出样式配置，因为模板的目的只是识别商品、SKU 和图片。
导出表头固定为：商品名称、销售属性1、SKU图片、销售属性2、数量。
页面只展示固定表头和最近监听批次的报货预览。
主页面不再展示 custom_product_text 等技术字段名，也不展示面单ID、面单模板等过程字段。
```

## 追加修正：报货 Excel 收敛为最终业务表

```text
用户明确最终报货表只需要“商品名称、销售属性1、SKU图片、销售属性2、数量”五列。
不管面单来源是我打、菜鸟直打还是抖店直打，识别阶段可以不同，但导出出口统一。
报货表第一张 Sheet 改为五列业务表，不再导出识别过程字段。
同一个商品/SKU/图片会合并为一行，销售属性2按出现顺序用空格合并，数量累加。
SKU图片列使用识别到的 SKU 绑定图片，插入 Excel 第三列。
商品或 SKU 未命中、冲突等异常不混入报货表，另建“异常明细” Sheet，避免静默漏单。
导出中心预览同步改为五列表，和下载的 Excel 第一张 Sheet 保持一致。
```
