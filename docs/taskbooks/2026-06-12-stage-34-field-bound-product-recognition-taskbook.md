# 阶段 34：按面单模式和字段绑定商品识别规则

## 背景

商品识别不能继续理解为“一个关键词匹配所有面单文字”。

目前平台已经可以读取三类面单信息：

```text
1. 抖店 / CloudPrint：平台可解析商品信息、商品简称、规格、数量、买家备注、卖家备注等结构化字段。
2. 菜鸟店铺直打：平台可解析 ITEM_INFO、买家备注、卖家备注等结构化字段。
3. 菜鸟 woda / 我打中转：标准面单字段加密，平台只能读取自定义打印区可见文字，字段含义由用户自己的打印模板决定。
```

因此客户侧商品识别规则必须绑定：

```text
面单模式
参与识别字段
关键词
目标商品
```

## 目标

```text
商品识别页支持选择面单模式。
商品识别页支持选择参与商品识别的字段。
抖店默认字段包含商品信息、商品简称、规格文本、买家备注、卖家备注。
菜鸟店铺直打默认字段包含商品信息、买家备注、卖家备注。
菜鸟 woda 默认字段为自定义打印文字。
保存商品识别规则时，把 mode_code 和 fields 写入 match_values。
识别预览按规则指定字段匹配关键词，不再默认匹配所有文本。
旧规则继续保留，但会显示为旧规则。
```

## 本阶段不做

```text
不做 woda 自定义字段拆分器。
不做复杂规则表达式、必须词、排除词。
不做识别结果回写数据库。
不做最终报货单生成闭环。
```
