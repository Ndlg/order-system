# 阶段 25：识别预览显式规则收紧

## 背景

识别预览页曾经存在一段兜底逻辑：

```text
如果没有命中客户维护的商品识别规则，就用商品名称本身去匹配面单文字。
```

这会导致一个问题：

```text
客户没有定义“5.0”作为识别关键词。
但只要商品库里存在名为“5.0”的商品，面单文字里出现“5.0”就会被自动识别为该商品。
```

该行为不符合当前产品原则：识别依据必须由客户显式定义，系统不应偷偷把商品名称当作规则。

## 目标

```text
移除商品名称兜底匹配。
识别预览只允许命中客户显式维护的数据。
命中来源需要能看清楚，避免误判系统如何识别。
```

## 已完成

```text
移除识别预览页中的商品名称兜底匹配。
保留“商品识别规则”命中。
接入“识别样本”的 product_keyword 命中。
命中信息显示来源：商品识别规则 / 识别样本。
更新识别步骤说明文案，明确商品识别来自规则或样本。
```

## 当前口径

```text
商品名称只是商品资料，不自动等同于识别规则。
商品识别规则是客户明确维护的“面单关键词 -> 商品”。
识别样本是客户明确标注的“面单原文片段 -> 商品/规格/尺码/数量”。
识别预览只使用商品识别规则和识别样本，不再使用商品名兜底。
```
