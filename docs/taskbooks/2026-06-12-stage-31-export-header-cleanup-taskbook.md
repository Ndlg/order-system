# 阶段 31：整理文档旧表头定义清理

## 背景

用户下载采集任务“整理”文档时，Excel 出现了以下列头：

```text
商品信息
商品简称
规格
尺码
数量
来源
辅助单号
```

这些列头并不是用户在当前客户管理页面明确配置的。

排查确认它们来自历史探索阶段写入当前工作区的 `field_definitions`，其中旧“识别预览”页面曾提供过“导出表头定义示例”和保存按钮。

## 问题

```text
客户没有明确配置表头，系统却输出了业务整理文档。
商品信息、商品简称、规格内容重复，容易让用户误解系统已完成报货单定义。
尺码字段来自系统推断，不是当前阶段客户明确配置的字段。
旧隐藏页面仍保留保存 field_definitions 的能力，后续可能再次写入错误表头。
```

## 目标

```text
没有客户明确配置导出表头时，不生成伪业务整理文档。
清理当前开发库中的历史导出字段。
移除旧隐藏页面保存表头定义的能力。
后续单独设计正式的报货单表头 / 导出字段定义页面。
```

## 已完成

```text
确认整理文档下载接口从 field_definitions 读取表头。
确认当前 Docker 数据库 workspace 1 有 7 条历史导出字段。
将这 7 条历史导出字段软删除并禁用：
  product_full_text
  product_display_text
  spec_text
  inferred_size
  quantity
  source_platform
  raw_document_id
移除旧识别预览页中的导出表头定义草稿、保存按钮和写入 field_definitions 的函数。
整理文档在没有导出表头时只输出提示，不再输出错误业务列。
提示文案改为“当前工作区还没有定义整理文档表头，暂不生成业务整理文档。”
```

## 当前口径

```text
原文文档可以继续用于诊断采集内容。
整理文档必须等客户明确配置导出表头后再生成。
当前抖店第一版先不把历史示例字段作为默认表头。
```
