# 阶段 27：抖店第一版客户端收口

## 背景

前期为了探索菜鸟、我打、自定义文字和复杂识别，客户管理端陆续加入了识别样本、识别预览、图片与档口等入口。

当前产品方向先收敛到抖店面单：

```text
抖店字段由平台端解析并固定字段类型。
客户端不需要定义抖店字段类型。
客户端只需要维护商品资料、SKU 图片和“抖店商品标题关键词 -> 我的商品”。
```

因此，客户管理端需要先恢复为清爽的抖店第一版流程，避免半成品入口干扰验收。

## 目标

```text
客户管理页只保留抖店第一版必需入口。
隐藏探索阶段入口。
旧地址不报错，统一重定向到当前主流程。
商品识别页面文案改为抖店商品标题关键词口径。
```
