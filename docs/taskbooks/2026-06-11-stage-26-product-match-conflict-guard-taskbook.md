# 阶段 26：商品识别冲突保护

## 背景

商品识别不能只做简单包含匹配。

典型问题：

```text
4.0 商品标题里也写了 5.0。
如果客户同时维护了 4.0 和 5.0 的商品识别规则，系统不能随机或按顺序选一个商品。
```

旧逻辑在多个关键词同时命中时，会按关键词长度和规则顺序选择一个结果。这会造成错误出单风险。

## 目标

```text
同一条面单文字同时命中多个不同商品时，不自动选择。
识别预览显示“商品冲突”。
冲突行不进入 SKU 自动匹配。
冲突行不显示为“可生成”。
页面列出冲突候选商品和命中关键词。
```

## 已完成

```text
识别预览页新增商品冲突检测。
商品识别规则和识别样本会先汇总所有命中候选。
同一商品命中多个关键词时，只保留该商品下最长的关键词作为候选说明。
多个不同商品同时命中时，结果标记为“商品冲突”。
商品冲突行不继续匹配 SKU。
商品冲突行状态显示为红色“商品冲突”。
结果筛选增加“商品冲突”选项。
商品识别列展示所有冲突候选：商品名、关键词、来源。
```

## 当前口径

```text
系统不能在 4.0 和 5.0 同时命中时替用户做决定。
冲突必须暴露给客户管理员处理。
后续需要继续补充规则能力：必须词、排除词、组合规则、人工确认回写样本。
```
