# 阶段 06 任务书：租户逻辑隔离与三端入口整改

> 适用范围：修正“平台后台、管理页面、业务页面”三类页面边界，并确认第一版数据库采用“单个平台库 + 多租户逻辑隔离”的实现路线。  
> 创建日期：2026-06-11  
> 状态：执行中

## 1. 背景

前一阶段已经完成登录、workspace 上下文、多用户数据隔离、用户工作台和开发者后台的初步拆分。

本阶段根据最新产品口径修正：

```text
系统不应只有“管理员端 / 用户端”两类页面。
系统应拆为三类页面：
1. 平台后台
2. 业务页面
3. 管理页面
```

追加端口边界：

```text
平台后台 UI 不能暴露给租户。
平台后台 UI 必须与租户端 UI 使用不同入口、不同端口和不同构建产物。
租户端 UI 只包含业务页面和管理页面。
```

同时确认第一版数据库路线：

```text
第一版采用单个平台数据库。
平台库中保存多个租户 / 客户、租户下的工作空间、用户与权限映射。
各租户业务数据通过 tenant_id + workspace_id 做逻辑隔离。
暂不为每个租户独立创建数据库。
```

## 2. 三端定义

### 2.1 平台后台

面向平台开发者 / 服务端管理员。

只维护平台控制面数据：

```text
客户 / 租户账户
客户工作空间映射
平台用户或客户主账号映射
当前支持的面单模式
面单模板
平台运行状态
```

明确不维护：

```text
采集器连接
采集任务
原始采集记录
用户字段定义
关键字段定义
图片资料
档口资料
匹配规则
业务异常
导出记录
```

### 2.2 管理页面

面向购买或使用平台的客户公司管理员。

负责维护本公司 / 本 workspace 的业务配置：

```text
字段定义
字段用途
关键字段选择
关键字段组合
图片资料
档口资料
匹配规则
导出配置
采集器连接
采集端授权或绑定
```

管理页面只能看到当前租户 / 当前 workspace 的数据。

### 2.3 业务页面

面向客户公司的一线业务员工。

只负责日常处理：

```text
采集面单信息
上传面单数据
查看采集记录
处理必要异常
导出结果
```

一线业务员工不需要理解平台底层资源表，也不直接维护面单模式、字段体系、图片档口定义等管理配置。

## 3. 数据库路线

### 3.1 当前选择

本阶段采用第一种方法：

```text
单个平台数据库
多租户逻辑隔离
tenant_id + workspace_id 作为数据归属边界
```

### 3.2 近期落地顺序

第一步先完成页面、构建产物和文档边界整改：

```text
租户端业务页面：http://127.0.0.1:5173/
租户端管理页面：http://127.0.0.1:5173/admin
平台后台 UI：http://127.0.0.1:5174/admin
```

第二步补齐租户层模型：

```text
tenants
workspaces.tenant_id
users 与 tenants / workspaces 的映射关系
业务表保留 workspace_id，并预留 tenant_id 收口策略
```

第三步补齐权限策略：

```text
platform_admin 只能进入平台后台
tenant_admin 进入管理页面
operator 进入业务页面
```

### 3.3 后续可扩展方向

未来如出现强隔离客户，可增加：

```text
租户数据库连接配置表
租户级数据源路由
独立库迁移工具
数据导入导出和迁移审计
```

但第一版不做每租户独立数据库，避免过早增加部署、迁移、备份和联调复杂度。

## 4. 本阶段交付范围

### 4.1 前端入口整改

新增或调整为两个 UI 应用入口：

```text
租户端 UI：/、/admin
平台后台 UI：5174 端口上的 /admin
```

租户端 UI 不得 import 平台后台路由。

保留兼容重定向，但不能把平台后台暴露到租户端：

```text
旧 /client -> /
旧 /client-admin -> /admin
旧 /workbench -> /
```

### 4.2 平台后台菜单

平台后台只允许出现：

```text
后台概览
客户 / 工作空间
用户
面单模式
面单模板
```

不得出现：

```text
采集连接
采集记录
字段定义
关键字段
图片与档口
匹配复核
异常处理
导出中心
```

### 4.3 管理页面菜单

管理页面应出现：

```text
管理首页
采集连接
字段定义
关键字段
图片与档口
匹配规则
```

### 4.4 业务页面菜单

业务页面应出现：

```text
业务首页
采集记录
面单批次
异常处理
导出中心
```

## 5. 验收标准

```text
1. 租户端登录后默认进入业务页面 `/`。
2. 租户端只显示管理页面、业务页面两个入口。
3. 租户端构建不 import 平台后台路由。
4. 平台后台使用独立私有端口。
5. 5174 端口上的 /admin 不出现任何客户业务数据维护入口。
6. 租户端 /admin 承载字段、关键字段、图片、档口、匹配规则和采集连接。
7. 租户端 / 承载采集记录、面单批次、异常处理和导出。
8. 旧 /workbench 可以重定向到 /。
9. 前端租户端构建通过。
10. 前端平台后台构建通过。
11. 后端现有隔离测试继续通过。
12. 文档同步记录本次整改，并保留旧阶段任务书。
```

## 阶段 07 追加勘误

根据后续确认的客户可见 URL 原则，租户端不再保留旧 `/client`、`/client-admin`、`/server-admin`、`/workbench` 兼容路由。客户可见端口只保留：

```text
/       业务页面
/admin  管理页面
```

平台私有后台仍在独立端口的 `/admin`，不能部署到客户可见端口。
