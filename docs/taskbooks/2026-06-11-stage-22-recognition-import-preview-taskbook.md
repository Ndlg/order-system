# 阶段 22：采集信息导入与识别预览页

> 阶段定位：在正式回写匹配结果前，先给客户管理员一个可视化预览页。用户可以导入已有采集任务的商品文字，也可以临时粘贴面单商品文字，系统即时预览商品规则命中、SKU 自动匹配和图片状态。

## 1. 目标

```text
从采集任务导入本轮 standard_details。
支持手动粘贴面单商品文字做临时预览。
按商品识别规则先匹配商品。
在命中的商品下面自动匹配 SKU。
展示 SKU 图片是否已绑定。
不在本阶段回写数据库，先用于验收识别效果。
```

## 2. 已完成

```text
新增客户管理页入口：/admin/recognition-preview。

客户管理首页新增“识别预览”卡片。
新增导入采集任务控件。
新增临时粘贴面单商品文字控件。
新增匹配概览：导入数、商品命中、SKU 命中、图片就绪。
新增预览表：面单商品文字、商品识别、SKU 自动匹配、图片预览、状态。
支持点击已命中 SKU 的图片做右侧预览。
```

## 3. 匹配口径

```text
商品识别：
  优先使用客户维护的商品识别规则。
  如果没有规则命中，临时用商品名称做兜底匹配。

SKU 自动匹配：
  只在已命中商品的 SKU 列表内匹配。
  优先完整命中 SKU 名称。
  其次用 SKU 名称拆分出的颜色、型号、尺码等 token 命中面单商品文字。

数量：
  优先读取平台解析出的 quantity。
  读不到时临时识别 “1 件” 或 “*1”。
```
