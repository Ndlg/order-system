# 阶段 30：采集控制页面单数口径修正

## 背景

客户业务页的“本轮采集控制”和“最近采集任务”原先显示的是原始采集记录数量。

这个数量容易被理解成面单数量，但实际更接近打印组件回传的记录数量。用户点击打印的次数、组件内部任务拆分、诊断记录都会影响它。

业务侧真正关心的是：

```text
这一轮采集最后整理出了多少张可处理面单。
```

## 目标

```text
采集控制页按钮位置稳定，不随状态和加载变化跳动。
“回传”口径从原始记录数改为面单数。
任务列表显示每个采集任务整理出的面单数量。
原始采集记录仍然保留在下方用于诊断，不作为业务主数量。
```

## 已完成

```text
采集记录页加载 standard_details。
按 standard_details.field_values.capture_task_id 统计每个采集任务的整理结果。
按 raw_document_id / logistics_no / order_no 去重统计面单数量。
本轮统计卡片由“本轮回传”改为“本轮面单”。
采集任务列表列名由“回传”改为“面单”。
本轮采集摘要由“回传 N 条”改为“面单 N 单”。
采集控制按钮改为固定 112px 宽度。
采集控制区新增独立布局，避免按钮因状态变化产生横向跳动。
```

## 当前口径

```text
面单数优先来自平台整理后的 standard_details。
原始 raw_capture_records 只用于诊断和原文下载。
如果采集到了原始记录但暂时无法整理为可处理面单，业务面单数显示为 0。
```
