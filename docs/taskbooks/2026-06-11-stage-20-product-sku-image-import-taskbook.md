# 阶段 20：商品父级与 SKU 图片导入整改

> 阶段定位：修正客户侧商品图片配置模型。商品不是从 ZIP 文件推断出来的，而是由客户先在管理页定义；ZIP 只作为该商品下面的 SKU 图片包导入。

## 1. 本阶段纠偏

原先理解偏差：

```text
上传 ZIP -> 系统从图片或文件名里推断商品 / SKU / 图片关系
```

正确业务关系：

```text
客户先创建商品名称
例如：匡威商品

客户选择这个商品
再上传这个商品对应的 SKU 图片 ZIP

系统把 ZIP 里的 SKU 图片导入到这个商品下面
形成：
商品 -> 多个 SKU -> 每个 SKU 绑定图片
```

## 2. 边界原则

```text
商品名称由客户定义，平台不从 ZIP 文件名自动决定商品名称。
SKU 属于商品子项，不能与商品平级。
图片绑定到 SKU，不直接替代商品。
后续面单商品文字先匹配商品，再匹配商品下的 SKU。
商品 / SKU 页面不维护匹配关键词，关键词映射商品 / SKU 应在专门匹配页面完成。
```


## 4. 当前 ZIP 支持口径

当前最小版本只处理已确认样例中的 SKU 图片目录：

```text
SKU图/
```

导入策略：

```text
只导入 SKU图 目录下的 jpg、jpeg、png、webp 图片。
详情图、主图、页面展示图暂不进入 SKU 匹配流程。
文件名只用于提取 SKU 名称，不写入匹配关键词。
商品名称始终以客户当前选中的商品为准。
```

例如：

```text
客户创建商品：匡威商品
上传 ZIP：
SKU图_05_木村-黑色.png

系统结果：
商品：匡威商品
SKU：黑色
图片：SKU图_05_木村-黑色.png
```
