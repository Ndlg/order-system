# 阶段 38：我打模板识别码与客户模板边界

> 日期：2026-06-12  
> 状态：已完成本轮最小修正

## 背景

我打 / woda 中转面单的标准面单内容在菜鸟组件中以 `encryptedData` 形式出现，但客户自定义打印区会以 `printXML` / `CDATA` 形式保存可见文字。

这说明平台不应该把该模式判断为“无法提供商品信息”，而应读取自定义打印区可见文字，并把字段含义交给客户工作区按自己的打印模板配置。

## 本轮目标

```text
1. 平台继续只负责读取和整理组件可见信息，不破解标准面单加密内容。
2. 我打模式把 printXML 中的可见文字保存为商品区原文。
3. 按 printXML 布局生成模板识别码，后续用于区分同一客户的不同打印模板。
4. 模板识别码只作为分组依据，不作为商品关键词匹配字段。
5. 客户模板含义由客户工作区后续单独维护，平台后台不维护客户业务模板。
```

## 已完成

```text
后端为 printXML 生成 print_template_key。
后端保存 print_template_source，用于说明模板识别来源。
后端保存 custom_area_lines，保留自定义区行文字。
菜鸟 woda 解析结果继续写入 product_full_text / custom_area_raw_text，保证业务页面可预览和后续匹配。
客户业务批次展开详情可展示模板识别码。
测试覆盖同一 printXML 布局生成同一模板识别码。
```

## 产品口径

```text
平台知道“这个面单属于哪种平台解析模式”。
平台可以按模板 URL 或 printXML 布局生成“模板识别码”。
平台不知道每个客户模板中每一行文字的业务含义。
客户管理员需要在自己的工作区里，按模板识别码查看样本并定义拆分规则。
同一个客户有多个模板时，系统按模板识别码分组配置。
不同客户即使模板识别码相同，规则也仍然按 tenant_id / workspace_id 隔离。
```

## 后续任务

```text
1. 新增客户管理端“打印模板规则”页面。
2. 按面单模式 + 模板识别码展示样本。
3. 允许客户定义：哪一行是商品文字、规格、尺码、数量、备注。
4. 支持规则试算，确认后保存到当前工作区。
5. 商品识别只使用用户确认后的业务字段，不直接全字段乱匹配。
```
