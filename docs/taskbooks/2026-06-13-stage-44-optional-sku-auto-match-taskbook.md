# 阶段 44：商品主类识别与 SKU 自动匹配

> 日期：2026-06-13  
> 状态：执行中，本轮修正商品识别规则口径

## 背景

阶段 43 已经补齐批次级商品/SKU 识别预览，但当前商品识别页仍然要求用户选择关联 SKU。实际业务中，一个商品主类下面可能有成百上千个 SKU，如果每个 SKU 都要维护一条识别规则，用户无法长期使用。

商品识别页应该主要解决“这条面单属于哪个商品主类”。SKU 应该在商品主类命中后，由系统根据面单销售属性和该商品下已有 SKU 自动匹配。

## 本轮目标

```text
商品识别页的关联 SKU 改为可选。
用户保存规则时，只要能识别商品主类即可。
商品主类可以通过商品文字识别，也可以通过销售属性1/销售属性2/备注字段识别。
后端识别引擎在命中商品后，自动用面单销售属性匹配该商品下的 SKU。
SKU 自动匹配优先读取销售属性1，其次销售属性2，最后备注字段。
保留用户显式选择 SKU 的能力，作为少量特殊规则的锁定命中。
```

## 识别流程

```text
商品行
  ↓
按用户选择的字段 + 商品主类关键词命中商品
  ↓
如果规则显式绑定 SKU，则使用该 SKU
  ↓
否则读取该商品下所有 SKU
  ↓
销售属性1 / 销售属性2 / 备注字段 对 SKU 名称和 SKU 关键词做自动匹配
  ↓
唯一命中则匹配成功
多命中则冲突
未命中则 SKU 未命中
```

## 业务边界

```text
1. 本阶段不要求用户逐 SKU 维护规则。
2. 本阶段不改变商品/SKU 数据模型。
3. SKU 自动匹配先使用 SKU 名称和 keywords 字段。
4. 如果 SKU 命名不规范，预览会显示 SKU 未命中或冲突，后续通过整理 SKU 名称/关键词解决。
5. 本阶段仍然只做预览，不写入 standard_details。
```

## 验收标准

```text
1. 商品识别页保存规则不再强制选择 SKU。
2. 保存未绑定 SKU 的规则时，target_type 为 product。
3. 批次识别预览中，命中商品后会自动匹配该商品下 SKU。
4. SKU 自动匹配支持销售属性1、销售属性2和备注字段。
5. 显式选择 SKU 时仍可保存为锁定 SKU 规则。
6. 后端测试通过。
7. 前端 tenant 构建通过。
8. 浏览器验证页面无控制台错误。
```

## 后续任务

```text
1. 在商品/SKU 管理页开放 SKU 关键词维护。
2. 识别预览中增加 SKU 自动匹配来源字段筛选。
3. 将识别结果持久化并接入导出 Excel。

```
