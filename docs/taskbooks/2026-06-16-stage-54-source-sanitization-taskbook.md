# 阶段 54：源码上传前去个人化清理

## 状态

已完成。

## 目标

将待上传源码目录整理为通用项目形态，避免夹带个人名称、现场地址、本机路径、业务现场专用 Docker 命名和用户数据。

## 处理范围

```text
部署 compose 文件
构建部署说明
服务器控制台脚本
平台客户账号示例文案
阶段记录中的现场专用命名
采集器下载包测试断言
```

## 调整内容

```text
1. 将站点部署 compose 文件改为 docker-compose.site.yml。
2. 将默认 Docker 项目、容器、镜像和数据卷命名收口为 order-system / order-system-* / order-system-data。
3. 构建部署说明改为通用占位符，不再写固定机器目录或现场地址。
4. 移除服务器控制台脚本中的本机固定 Python 路径。
5. 平台客户账号示例改为通用示例客户。
6. 历史阶段记录中的现场专用路径、地址和命名改为占位写法。
7. 测试断言从固定地址字符串改为通用私有网段防泄露校验。
```

## 验证结果

