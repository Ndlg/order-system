# 平台职责边界

> 本文档是辅助原则文档。如与 `docs/CODEX_TASKBOOK.md` 冲突，以 `docs/CODEX_TASKBOOK.md` 为准。

## 最终原则

系统要做的是平台框架，不是业务规则定制。

一句话：

```text
系统只读取和处理面单，业务字段和匹配规则都交给用户定义。
```

## 系统负责什么

系统负责提供通用平台能力：

```text
用户 / 工作空间 / 权限
面单模式管理
面单解析模板管理
采集端和采集任务框架
原始采集内容留存
标准化明细管理
面单字段定义器
关键字段配置器
关键字段匹配规则维护器
图片信息管理器
档口资料维护器
报货汇总引擎
Excel 导出引擎
异常记录与人工修正流程
操作日志
```

## UI 入口与端口边界

系统前端必须区分两个 UI 应用入口：

```text
租户端 UI
平台后台 UI
```

租户端 UI 可以暴露给客户，包含两个页面区域：

```text
业务页面：采集记录、面单批次、异常处理、导出中心。
管理页面：采集连接、字段定义、关键字段、图片、档口、匹配规则、导出配置。
```

平台后台 UI 面向平台开发者 / 服务端管理员，只维护平台控制面：

```text
客户 / 租户账户
工作空间映射
用户
当前支持的面单模式
平台共用面单解析模板
```

平台后台 UI 的端口不能暴露给租户，必须使用独立端口、内网、本机或独立部署单元。

采集器连接、采集任务、原始采集记录、字段定义、图片、档口、报表、异常和导出数据都属于租户工作空间数据，应在业务页面或管理页面维护。

## 系统不负责什么

系统代码不应该知道：

```text
哪个字段代表哪种业务含义
哪些字段必须存在
哪些字段参与图片匹配
哪些字段参与档口匹配
哪些字段用于汇总分组
哪个关键字段组合对应哪张图片
哪个关键字段组合对应哪个档口
最终 Excel 必须按哪一种业务固定格式输出
```

这些全部由用户在 Web 界面配置。

## 面单模式是系统核心入口

系统识别出面单模式后，只做平台化处理：

```text
面单模式
  ↓
套用平台共用解析模板
  ↓
生成客户可读面单信息
  ↓
用户定义面单字段
  ↓
用户选择关键字段
  ↓
用户维护图片 / 档口匹配规则
  ↓
用户生成报货 Excel
```

## 用户定义内容

以下内容必须由用户维护，不允许写死在代码里：

```text
面单字段定义
关键字段组合
图片匹配规则
档口匹配规则
数量统计方式
备注保留方式
导出字段
导出分组方式
导出文件拆分方式
```

## 平台默认值

允许系统提供默认模板和默认导出配置，便于第一版快速使用。

但默认值必须满足：

```text
1. 默认值写入数据库初始化数据或可编辑配置。
2. 默认值可以被用户修改或替换。
3. 默认值不能写死在业务代码中。
4. 默认值不能成为唯一不可变规则。
```

## P0 开发边界

第一阶段 P0 的目标不是写死任何业务规则，而是完成平台闭环：

```text
用户上传 Excel
  ↓
识别 / 选择面单模式
  ↓
读取面单字段
  ↓
用户定义字段和关键字段
  ↓
用户配置图片 / 档口匹配规则
  ↓
生成报货汇总
  ↓
按用户选择的导出方式导出 Excel
```

## 禁止事项

```text
禁止在代码里写死具体用户业务规则。
禁止在代码里写死具体面单字段。
禁止在代码里写死图片匹配规则。
禁止在代码里写死档口匹配规则。
禁止在代码里写死固定报货分组规则。
禁止让系统依赖某一种固定面单字段。
禁止绕过用户配置直接生成报货结果。
```
