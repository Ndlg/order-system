# 平台职责边界

> 本文档是辅助原则文档。如与 `docs/CODEX_TASKBOOK.md` 冲突，以 `docs/CODEX_TASKBOOK.md` 为准。

## 最终原则

系统要做的是平台框架，不是业务规则定制。

一句话：

```text
系统只识别和处理面单模式，其他业务规则都交给用户定义。
```

## 系统负责什么

系统负责提供通用平台能力：

```text
用户 / 工作空间 / 权限
面单模式管理
采集端和采集任务框架
原始采集内容留存
面单模板管理
标准化明细管理
字段角色配置器
映射表维护器
图片文件管理器
档口资料维护器
报货汇总引擎
Excel 导出引擎
异常记录与人工修正流程
操作日志
```

## 系统不负责什么

系统代码不应该知道：

```text
哪个字段一定是店铺
哪个字段一定是鞋款简称
哪个简称对应哪个鞋款
哪个规格对应哪个标准规格
哪个鞋款归哪个档口
哪个规格用哪张图片
最终 Excel 必须按哪一种业务固定格式输出
```

这些全部由用户在 Web 界面配置。

## 面单模式是系统核心入口

系统识别出面单模式后，只做平台化处理：

```text
面单模式
  ↓
套用该面单模式下的模板配置
  ↓
生成标准化明细
  ↓
用户选择字段角色
  ↓
用户维护映射关系
  ↓
用户生成报货 Excel
```

## 用户定义内容

以下内容必须由用户维护，不允许写死在代码里：

```text
字段角色配置
业务简称 / 鞋款简称映射
规格映射
图片绑定
档口归属
数量统计方式
尺码展示方式
导出字段
导出分组方式
导出文件拆分方式
```

## 平台默认值

允许系统提供默认模板和默认导出配置，便于第一版快速使用。

但默认值必须满足：

```text
1. 默认值写入数据库初始化数据或可编辑配置。
2. 默认值可以被用户修改或替换。
3. 默认值不能写死在业务代码中。
4. 默认值不能成为唯一不可变规则。
```

## P0 开发边界

第一阶段 P0 的目标不是写死鞋款业务，而是完成平台闭环：

```text
标准化 Excel 上传
  ↓
识别 / 选择面单模式
  ↓
读取标准化明细
  ↓
用户配置字段角色
  ↓
用户配置映射关系
  ↓
用户配置图片 / 档口
  ↓
生成报货汇总
  ↓
按用户选择的导出方式导出 Excel
```

## 禁止事项

```text
禁止在代码里写死店铺规则。
禁止在代码里写死鞋款简称规则。
禁止在代码里写死规格规则。
禁止在代码里写死档口归属规则。
禁止在代码里写死图片绑定规则。
禁止在代码里写死固定报货分组规则。
禁止在代码里把店铺作为整理主线。
禁止让系统依赖某一种固定面单字段。
禁止绕过用户配置直接生成报货结果。
```
