# 前端入口与受众边界

> 本文档说明前端入口、受众和端口暴露边界。如与 `docs/CODEX_TASKBOOK.md` 冲突，以 `docs/CODEX_TASKBOOK.md` 为准。

## 核心结论

客户可见地址不使用 `client` 或 `server` 字样。

租户端不保留 `/client`、`/client-admin`、`/server-admin`、`/workbench` 作为客户可见兼容入口。

租户端 UI 对客户开放：

```text
默认地址 / 进入业务页面
/admin 进入管理页面
```

平台后台 UI 不对客户开放，使用独立私有端口：

```text
http://127.0.0.1:5174/admin
```

## 端口边界

开发环境端口：

```text
租户端 UI：http://127.0.0.1:5173/
租户端管理：http://127.0.0.1:5173/admin
平台后台 UI：http://127.0.0.1:5174/admin
```

部署原则：

```text
1. 对客户开放的端口只发布业务页面和管理页面。
2. 平台后台端口不能挂在公网租户域名下。
3. 平台后台构建产物不能混入租户端构建产物。
4. 平台后台必须通过内网、防火墙、VPN、堡垒机或本机访问策略保护。
```

## 业务页面

面向客户公司一线员工。

页面主线：

```text
查看采集记录
上传 / 读取面单
处理必要异常
导出结果
```

业务页面应隐藏或弱化：

```text
平台控制面
租户账户维护
用户和工作空间映射
面单解析规则底层维护
字段体系管理
图片 / 档口资料维护
匹配规则配置
OpenAPI / 开发调试信息
```

## 管理页面

面向客户公司管理员。

负责维护当前租户 / 当前 workspace 的业务配置：

```text
采集连接
字段定义
字段用途
关键字段
图片资料
档口资料
匹配规则
导出配置
```

管理页面只能看到当前租户 / 当前 workspace 的数据，不能看到其他客户数据，也不能进入平台后台控制面。

## 平台后台 UI

面向平台开发者 / 服务端管理员。

只维护平台控制面数据：

```text
客户 / 租户账户
工作空间映射
用户
平台共用面单解析模式
平台共用面单解析模板
```

平台后台 UI 明确不维护：

```text
采集器连接
采集任务
原始采集记录
用户字段定义
关键字段组合
图片资料
档口资料
匹配规则
业务异常
导出记录
```

这些都属于租户工作空间数据，归业务页面或管理页面处理。

## 设计逻辑

系统不能先替用户预设业务概念。正确顺序是：

```text
系统先读取面单并按平台共用规则整理成可读面单信息
  ↓
客户管理员定义这些面单信息的字段用途
  ↓
客户管理员决定关键字段、图片、档口和匹配规则
  ↓
一线员工在业务页面采集、上传、异常处理和导出
```

## 开发要求

后续前端开发必须满足：

```text
1. 租户端 UI 不 import 平台后台路由。
2. 平台后台 UI 使用独立入口和独立端口。
3. 平台后台 UI 不出现在租户端入口切换中。
4. 业务页面不维护字段体系、图片、档口和匹配规则。
5. 管理页面负责当前 workspace 的业务配置。
6. 采集器绑定、连接配置和状态维护由管理页面负责，不进入平台后台。
7. 开始采集、结束采集和本轮采集记录查看由业务页面负责，便于统一控制业务机采集器。
8. 字段含义、关键字段、图片、档口、导出方式全部由客户配置产生。
9. 租户端路由不暴露旧 `client`、`server`、`workbench` 命名入口。
```
